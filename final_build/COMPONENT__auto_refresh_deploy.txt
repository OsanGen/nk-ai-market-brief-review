# Component: Auto Refresh Deploy

## Purpose

Refresh the NK AI Market Brief page automatically through GitHub Actions and optional GitHub Pages so normal weekday updates do not require local builds or manual pushes after repository setup.

## Inputs

- `.github/workflows/newsletter.yml`
- `newsletter-sources.json`
- RSS metadata fetched by the newsletter pipeline
- Repository variable `DEPLOY_GITHUB_PAGES`
- Optional email variables, kept disabled by default

## Outputs

- Scheduled GitHub Actions run history
- GitHub Actions artifact `nk-ai-market-brief`
- Optional GitHub Pages deployment of generated `site/`
- Automation metadata in `site/run.json`
- Automation status section in `site/index.html`
- Live Pages URL: `https://osangen.github.io/nk-ai-market-brief-review/`

## Schedule Behavior

- Manual `workflow_dispatch` remains available.
- Weekday schedules:
  - `17 12 * * 1-5`
  - `17 13 * * 1-5`
- The workflow runs from the latest default branch commit.
- The runtime guard targets 8 a.m. America/New_York for auto mode.

## GitHub Pages Behavior

- Workflow includes `actions/configure-pages@v5`, `actions/upload-pages-artifact@v3`, and `actions/deploy-pages@v4`.
- Pages deployment is gated by repository variable `DEPLOY_GITHUB_PAGES=true`.
- Repository Settings -> Pages must use Source: GitHub Actions.
- The public Pages repository is configured with Pages build type `workflow` and repository variable `DEPLOY_GITHUB_PAGES=true`.

## Artifact Fallback Behavior

If Pages is not enabled, the workflow still uploads `.newsletter-outbox` and `site` as the `nk-ai-market-brief` artifact.

## Email Disabled Behavior

The default workflow sets `NEWSLETTER_SEND_ENABLED=false`. Email cannot send from scheduled refresh by default.

The final shareable review page still records send state in `site/run.json` and the lower Automation status section, but no workflow or local preview run sends email unless the explicit send gates are configured outside the default preview path.

## Acceptance Checks

- `npm install`
- `npm run preview`
- `npm test`
- `npm run build`
- `npm run check:deploy`
- `site/run.json` has `automationConfigured: true`
- `site/run.json` has `manualPushRequiredAfterSetup: false`
- Workflow has manual dispatch, both cron entries, artifact upload, Pages upload/deploy, and Pages permissions.
- `site/index.html` shows Automation status below the selected signals and technical diagnostics.
- GitHub Pages deploy support remains gated by `DEPLOY_GITHUB_PAGES=true`; artifact upload remains available independently.

## Remaining Setup Steps

- Keep email disabled until sender, recipients, and explicit approval are provided.
- Monitor GitHub's Node 20 action runtime deprecation warning and move actions to Node 24-compatible versions if needed.
