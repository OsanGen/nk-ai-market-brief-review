# Component: Auto Refresh Deploy

## Purpose

Refresh the NK AI Market Brief page automatically through GitHub Actions and optional GitHub Pages every calendar day around 4 a.m. America/New_York so normal updates do not require local builds or manual pushes after repository setup.

## Inputs

- `.github/workflows/newsletter.yml`
- `newsletter-sources.json`
- RSS metadata fetched by the newsletter pipeline
- Repository variable `DEPLOY_GITHUB_PAGES`
- Optional email variables, kept disabled by default

## Outputs

- Scheduled GitHub Actions run history
- GitHub Actions artifact `nk-ai-market-brief`
- Optional GitHub Pages deployment of generated `site/`
- Automation metadata in `site/run.json`
- Automation status section in `site/index.html`
- Live Pages URL: `https://osangen.github.io/nk-ai-market-brief-review/`

## Schedule Behavior

- Manual `workflow_dispatch` remains available.
- Daily 4 a.m. retry schedules:
  - `2,7,12,17,22,27,32,37,42,47,52,57 8,9 * * *`
  - `17 10,11,12 * * *`
- The 8/9 UTC schedule covers daylight saving and standard-time 4 a.m. Eastern without hitting exactly `:00`.
- The 10/11/12 UTC schedule is a late watchdog recovery layer.
- The workflow runs from the latest default branch commit.
- `npm run should:refresh` checks the live `run.json` before dependency install on scheduled `auto` runs.
- Scheduled retries skip if the live page is already fresh for today in America/New_York.
- If the live freshness check cannot be fetched, the gate fails open and runs the refresh.
- Scheduled/public runs use `npm run daily`, which forces `auto` mode so delayed GitHub cron starts still publish a generated daily page.
- Daily public output uses `NEWSLETTER_LOOKBACK_HOURS=36` on normal days and `NEWSLETTER_MONDAY_LOOKBACK_HOURS=84` on Mondays.
- Daily freshness beats perfect story count: expected daily deploy checks allow limited-story output as long as the page is current, auto-mode, and within the daily lookback ceiling.
- Manual preview dispatch still uses `npm run build` and the broader 168-hour review lookback.

## GitHub Pages Behavior

- Workflow includes `actions/configure-pages@v5`, `actions/upload-pages-artifact@v3`, and `actions/deploy-pages@v4`.
- Pages deployment is gated by repository variable `DEPLOY_GITHUB_PAGES=true`.
- Repository Settings -> Pages must use Source: GitHub Actions.
- The public Pages repository is configured with Pages build type `workflow` and repository variable `DEPLOY_GITHUB_PAGES=true`.

## Artifact Fallback Behavior

If Pages is not enabled, the workflow still uploads `.newsletter-outbox` and `site` as the `nk-ai-market-brief` artifact.

## Live Freshness Behavior

- `scripts/check-live-freshness.mjs` fetches `https://osangen.github.io/nk-ai-market-brief-review/run.json` with cache busting after Pages deploy.
- It retries before failing the workflow.
- It passes only when `generatedAt` is today in America/New_York, `mode` is `auto`, `config.activeLookbackHours <= 84`, `automationConfigured` is true, and `send.sent` is false.

## Email Disabled Behavior

The default workflow sets `NEWSLETTER_SEND_ENABLED=false`, and `npm run daily` also disables sending for public page generation. Email cannot send from scheduled refresh by default.

The final shareable review page still records send state in `site/run.json` and the lower Automation status section, but no workflow or local preview run sends email unless the explicit send gates are configured outside the default preview path.
