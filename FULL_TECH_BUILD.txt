## 2. What this repo is

This repo is a free, standalone internal NK AI Market Brief MVP. It produces a weekday market brief focused only on AI plus fashion, beauty, e-commerce, agentic commerce, AI shopping, and AI agents or assistants used by major fashion, beauty, retail, and commerce platforms or brands. The default mode uses Google News RSS metadata, deterministic filtering/scoring, local/static outputs, GitHub Actions artifacts, no secrets, no database, no LLM, and no page scraping.

User-facing artifacts:
- Local preview: `npm run preview`
- Public daily refresh: `npm run daily`
- Build static site: `npm run build`
- Deploy check: `npm run check:deploy`
- Optional send: `npm run send`
- Outputs: `.newsletter-outbox/YYYY-MM-DD/newsletter.html`, `.newsletter-outbox/YYYY-MM-DD/newsletter.txt`, `.newsletter-outbox/YYYY-MM-DD/run.json`, `site/index.html`, `site/newsletter.txt`, `site/run.json`
- GitHub repository: `https://github.com/OsanGen/nk-ai-market-newsletter` (private)
- Live review page: `https://osangen.github.io/nk-ai-market-brief-review/`
- Public workflow-capable Pages repo: `https://github.com/OsanGen/nk-ai-market-brief-review`
- Cyril evidence pack: `CYRIL_VERIFICATION_PACK.md`
- Share instructions: `SHARE_WITH_CYRIL.md`

## 3. System map

Components:
- `newsletter-sources.json`: nine original source definitions from the NK context plus three review-yield market scan source definitions.
- `src/config.mjs`: no-secret defaults, comma-separated recipient parsing, optional env parsing, daily lookback windows, and review-mode lookback.
- `src/sources.mjs`: source registry validation and disabled-source filtering.
- `src/google-news.mjs`: Google News RSS URL construction.
- `src/fetch-feeds.mjs`: RSS fetch/parse with `rss-parser`, built-in `fetch`, 15-second source timeout, and source-level error capture.
- `src/normalize.mjs`: normalized RSS item shape, actual RSS outlet extraction, tag stripping, display-text neutralization, escaping helpers, and URL safety.
- `src/filter-score.mjs`: strict title/summary inclusion/exclusion using word/phrase-boundary matching so source categories or substrings like `ai` inside `retail`/`openai` cannot admit generic news.
- `src/dedupe.mjs`: canonical URL and title-overlap dedupe.
- `src/story-diversity.mjs`: review story selection that caps repeated topic clusters at two when enough qualifying items exist and prefers source/category diversity for close scores.
- `src/summarize.mjs`: deterministic RSS-only headline, outlet suffix cleanup, non-duplicative summary fallback, issue-level copy rotation, business-facing why-it-matters, and category fields.
- `src/render-html.mjs` and `src/render-text.mjs`: escaped/neutralized email-style HTML and text rendering.
- `src/review-status.mjs`: review readiness and limited-yield state from selected item count.
- `src/automation-status.mjs`: workflow automation detection for schedule, artifact upload, Pages deploy support, and manual-push status.
- `src/render-review-page.mjs`: escaped static review page rendering for `site/index.html`, including noindex metadata, status badges, review status, selected-signal label, lead story, story grid when available, actual outlet source labels, scan labels, visually secondary technical diagnostics, source fetch status, automation status, and `newsletter.txt` link.
- `src/send-resend.mjs`: optional Resend HTTP API send gate, disabled by default, minimum-item check, recipient validation, and idempotency header.
- `src/time-guard.mjs`: weekday 8 a.m. America/New_York guard.
- `src/run-newsletter.mjs`: orchestrator and output writer.
- `scripts/*.mjs`: npm script entrypoints for preview, public daily refresh, build, explicit send, and deploy checks.
- `.github/workflows/newsletter.yml`: manual/scheduled CI, tests, daily public output for scheduled `auto`, review build for manual preview, deploy checks, artifact upload, optional Pages deployment gated by `DEPLOY_GITHUB_PAGES=true`.

Data flow: enabled sources -> Google News RSS URLs -> RSS metadata -> normalized items with outlet source -> strict filter/score -> dedupe -> story-diversity selection in preview mode or top-score daily selection in auto mode -> deterministic summaries -> review-readiness and automation-status calculation -> neutralized newsletter HTML/text plus static review page -> outbox + site -> deploy check enforcing `reviewReady`, scheduled automation, and expected daily mode/lookback when configured -> artifact upload and optional GitHub Pages -> optional gated send outside the default public workflow.

## 4. Entrypoints and run commands

Required commands:
```bash
npm install
npm run preview
npm run daily
npm test
npm run build
npm run check:deploy
NEWSLETTER_EXPECT_MODE=auto NEWSLETTER_MAX_ACTIVE_LOOKBACK_HOURS=84 npm run check:deploy
```

Explicit send failure proof:
```bash
NEWSLETTER_SEND_ENABLED=true npm run send || true
```

Shareable review page proof:
```bash
npm run build
npm run check:deploy
```

Optional send command:
```bash
npm run send
```

Environment:
- Default mode requires no secrets.
- `.env.example` documents optional email settings and defaults to `NEWSLETTER_SEND_ENABLED=false`.
- Preview/build mode uses `NEWSLETTER_REVIEW_LOOKBACK_HOURS=168` by default so the review page can show enough qualifying signals. Scheduled `auto` mode keeps `NEWSLETTER_LOOKBACK_HOURS=36` and `NEWSLETTER_MONDAY_LOOKBACK_HOURS=84`.
- `npm run daily` is the public scheduled page path: it forces `auto` mode, keeps sending disabled, and uses the normal daily lookback windows so delayed GitHub cron starts still publish generated output.
- Preview mode cannot send.
- Email cannot send unless all gates pass: `NEWSLETTER_SEND_ENABLED=true`, send/auto mode, `RESEND_API_KEY`, `NEWSLETTER_FROM`, valid `NEWSLETTER_TO`, and enough items.
- Default GitHub Actions workflow runs from the default branch on weekday schedules at `17 12 * * 1-5` and `17 13 * * 1-5`, supports manual dispatch with `mode`, uses `npm run daily` plus mode/lookback deploy guards for scheduled public output, supports Pages deploy when `DEPLOY_GITHUB_PAGES=true`, and keeps email disabled by default.
