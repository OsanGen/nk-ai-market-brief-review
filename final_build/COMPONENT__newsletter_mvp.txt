# Component: Newsletter MVP

## Purpose

Generate a weekday internal NK AI Market Brief focused only on AI plus fashion, beauty, e-commerce, agentic commerce, AI shopping, and AI agents or assistants used by major fashion, beauty, retail, and commerce platforms or brands.

## Inputs

- `NK_NEWSLETTER_MVP_CONTEXT.txt`
- `newsletter-sources.json`
- Optional environment variables listed in `.env.example`
- Google News RSS responses

## Outputs

- `.newsletter-outbox/YYYY-MM-DD/newsletter.html`
- `.newsletter-outbox/YYYY-MM-DD/newsletter.txt`
- `.newsletter-outbox/YYYY-MM-DD/run.json`
- `site/index.html`
- `site/newsletter.txt`
- `site/run.json`
- GitHub Actions artifact: `nk-ai-market-brief`
- Cyril evidence pack: `CYRIL_VERIFICATION_PACK.md`
- Cyril share instructions: `SHARE_WITH_CYRIL.md`

## State

No database and no durable remote state. Every run creates a dated outbox and refreshes the latest static `site/` output.

Preview/build mode is review-oriented and uses `NEWSLETTER_REVIEW_LOOKBACK_HOURS` with a 168-hour default so the share page can demonstrate enough current signals. The public daily path uses `npm run daily`, forces `auto` mode, and keeps the normal daily windows: `NEWSLETTER_LOOKBACK_HOURS=36` and `NEWSLETTER_MONDAY_LOOKBACK_HOURS=84`.

## Edge Cases

- One source can fail without failing the whole run.
- Preview/build work without secrets.
- Auto mode outside the target window writes a safe static skip state and exits successfully.
- Send mode fails safely when send gates are not met.
- No raw feed HTML is rendered.
- Rejected raw items are not exposed in public output.
- Generic fashion, beauty, e-commerce, AI, retail, and platform stories are rejected unless the story title/summary itself contains an allowed AI-plus-market signal.
- Preview mode cannot send, even if send env vars are present.
- Auto mode without secrets generates artifacts and records a skipped send.
- Explicit send without required env vars exits safely with a skipped reason and no secret values.
- Malicious feed text cannot inject `<script>`, event handler attributes, or `javascript:` URLs into generated HTML/text.
- GitHub Actions preview runs upload generated outbox and site files as an artifact without enabling email by default.
- QA evidence is captured in `final_build/QA__newsletter_mvp.txt` and the Cyril-facing share pack.
- `site/index.html` is a static review page, while `.newsletter-outbox/YYYY-MM-DD/newsletter.html` remains the email-style newsletter HTML.
- Review readiness is recorded in `site/run.json`; `npm run check:deploy` fails when `reviewReady` is false unless `ALLOW_NOT_READY_REVIEW=true` is set for debugging.
- Automation readiness is recorded in `site/run.json`; `npm run check:deploy` fails when scheduled refresh or workflow-backed Pages support is missing.
- Public daily deploy checks can require `NEWSLETTER_EXPECT_MODE=auto` and `NEWSLETTER_MAX_ACTIVE_LOOKBACK_HOURS=84` so preview-mode review output is not accidentally deployed as the daily page.
- Review story selection uses `src/story-diversity.mjs` to avoid over-representing repeated clusters such as Meta shopping agents, Google virtual try-on, OpenAI commerce, Shopify catalog/storefronts, beauty AI discovery, fashion AI discovery, and agentic commerce platform race. It does not drop valid items below the minimum review threshold only for diversity.
- Normalized stories carry both the actual RSS outlet source and the configured scan label. Renderers show the outlet as `Source` and the query family as `Scan` so configured scans are not mistaken for publications.
- Deterministic summary copy avoids repeating the same summary or why-it-matters sentence more than twice in one issue and keeps implementation-language filter explanations out of client-facing output.

## Dependencies

- Node 20+
- `rss-parser`
- Built-in `fetch`
- Built-in `node:test`

## Acceptance Checks

- `npm install`
- `npm run preview`
- `npm run daily`
- `npm test`
- `npm run build`
- `npm run check:deploy`
- `NEWSLETTER_EXPECT_MODE=auto NEWSLETTER_MAX_ACTIVE_LOOKBACK_HOURS=84 npm run check:deploy`
- `NEWSLETTER_SEND_ENABLED=true npm run send || true`
- `gh workflow run newsletter.yml -f mode=preview --ref main`
- `env -u RESEND_API_KEY -u NEWSLETTER_FROM -u NEWSLETTER_TO NEWSLETTER_SEND_ENABLED=true npm run send || true`
- Unsafe-output grep has no matches for secrets, scripts, event handlers, `javascript:` URLs, `metadata matched`, or `matched filters`.
