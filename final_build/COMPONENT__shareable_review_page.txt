# Component: Shareable Review Page

## Purpose

Provide a static, no-login review page for Cyril, Faneeza, and Norma to inspect the latest NK AI Market Brief output before email delivery is enabled.

## Inputs

- Story summaries generated by `src/summarize.mjs`
- Run metadata from `src/run-newsletter.mjs`
- Source result metadata from Google News RSS fetches
- Safety helpers from `src/normalize.mjs`
- Review-readiness metadata from `src/review-status.mjs`
- Automation metadata from `src/automation-status.mjs`

## Outputs

- `site/index.html`: shareable review page
- `site/newsletter.txt`: plain-text companion
- `site/run.json`: machine-readable run summary
- GitHub Actions artifact: `nk-ai-market-brief`
- `SHARE_WITH_CYRIL.md`: handoff instructions and copy/paste message
- Live URL: `https://osangen.github.io/nk-ai-market-brief-review/`
- Public workflow-capable repository: `https://github.com/OsanGen/nk-ai-market-brief-review`

## GitHub Pages Behavior

The source workflow always builds `site/` and uploads it as an artifact. The private source repository's current GitHub plan does not support Pages, so the live page is hosted from a public workflow-capable repo that contains the no-secret app and deploys generated `site/` through GitHub Actions. The page includes `<meta name="robots" content="noindex,nofollow">` because it is a review page, not a public SEO page.

The page is shareable only when `site/run.json` has `"reviewReady": true`. Review mode targets 6-8 selected stories when enough qualifying RSS items exist; if fewer than 5 stories are selected, the page labels itself as needing source tuning. If fewer than 3 are selected, deploy check fails unless `ALLOW_NOT_READY_REVIEW=true` is set for debugging.

The page includes a lower Automation status section showing scheduled refresh, GitHub Pages support, artifact fallback, and email-disabled status.

The selected-signal area prioritizes the market brief content. Each story card shows the actual RSS outlet as `Source`, the primary category, the configured source-query family as `Scan`, date when available, a deterministic summary, business-facing why-it-matters copy, and a `Read source` link. The renderer keeps Technical diagnostics, Source fetch status, and Automation status visually secondary and below the story content.

## Artifact Fallback

If Pages is unavailable, reviewers can use the latest GitHub Actions artifact:

`Actions -> NK AI Market Brief -> latest run -> artifact nk-ai-market-brief`

## Safety Rules

- Static HTML only.
- No external CSS, JavaScript, images, tracking, login, or database.
- Escape all feed-provided text.
- Never render raw RSS HTML.
- Never expose secrets, recipients, API keys, or local-only environment values.
- Email remains disabled by default.
- Do not show an empty story grid as ready; if there are too few additional qualifying stories, show source/filter tuning language instead.
- Do not expose implementation-language filter explanations such as `metadata matched` or `matched filters`.

## Acceptance Checks

- `npm run build`
- `npm run check:deploy`
- `npm test`
- `site/index.html` includes `NK AI Market Brief`, `Internal review`, `Email disabled`, `newsletter.txt`, and noindex metadata.
- `site/run.json` includes `reviewReady: true`, `minReviewItems`, `candidateItemCount`, `selectedItemCount`, `sourceErrorCount`, and `sendStatus`.
- `site/run.json` includes `automationConfigured: true`, `scheduledRefreshConfigured: true`, `githubPagesDeployConfigured: true`, and `manualPushRequiredAfterSetup: false`.
- Review output selects at least five stories when enough qualifying feed items exist and limits repeated topic clusters to two.
- Story labels visibly distinguish `Source` from `Scan`.
- Technical diagnostics, source fetch status, and automation status remain present but visually secondary.
- Unsafe-output grep has no matches for secrets, scripts, event handlers, or `javascript:` links.
